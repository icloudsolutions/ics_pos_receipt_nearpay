ICS POS Receipt NearPay

Native NearPay Transaction Details in Standard Odoo 18 POS Receipt

Version 18.0.1.0.0

📋
Overview

The ICS POS Receipt NearPay module extends the standard Odoo 18 Point of Sale receipt to display NearPay transaction details directly in the receipt footer. Unlike image-based approaches, this solution uses Odoo's native receipt rendering system for perfect compatibility with all printers (ESC/POS, browser, mobile).

This lightweight module integrates seamlessly with your existing NearPay payment flow and requires no additional dependencies or image generation.

Integration Points:

  • Models: Extends PosOrder.export_for_printing() to inject NearPay data
  • Template: Uses after-footer slot in standard receipt
  • Settings: Adds configurable toggle in POS config
  • Compatibility: Works with any terminal payment method storing NearPay fields

📋
Requirements

  • Odoo Community/Enterprise 18.0 with Point of Sale installed
  • NearPay Payment Integration that stores transaction data on pos.payment records (e.g., nearpay_transaction_id, nearpay_approval_code, etc.)
  • Payment Method configured with payment_method_type = 'terminal'

⚙️
Installation & Configuration

  1. Download and place the ics_pos_receipt_nearpay folder in your Odoo addons directory
  2. Update the apps list and install "ICS POS Receipt NearPay"
  3. Go to Point of Sale > Configuration > Settings
  4. Enable "Show NearPay Info on Receipt" under Payment section
  5. Process a NearPay payment — transaction details will appear in the receipt footer

NearPay Data Fields (must be present on payment record):

# Required fields (at minimum): - nearpay_transaction_id - nearpay_approval_code # Optional but recommended: - nearpay_card_scheme - nearpay_status_message - nearpay_terminal_id - nearpay_retrieval_ref_no - nearpay_currency - nearpay_amount_authorized

🔄
Workflow Process

1. Payment Processing

NearPay transaction is processed via your existing Flutter/terminal integration.

2. Data Storage

Transaction details are saved on the pos.payment record in Odoo POS.

3. Receipt Generation

When printing, export_for_printing() includes NearPay data in receipt context.

4. Native Rendering

Receipt template displays NearPay info in the footer using standard Odoo rendering.

5. Universal Printing

Receipt prints correctly on any device or printer without workarounds.
